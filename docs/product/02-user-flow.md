# User Flow

## Main journey

```txt
Visitor
  -> Landing page
  -> Safety and consent onboarding
  -> Guided AI conversation
  -> Risk-aware support routing
  -> Clarity Map generation
  -> Resources and next steps
  -> Feedback
```

## Screen 1: Landing

Purpose: make the promise obvious.

Headline:

> Understand what you are going through. Find the right kind of support.

Subheadline:

> Talk with an AI clarity assistant that helps you organize what you are experiencing, identify focus areas, and decide what kind of support may be useful.

CTA:

> Start a clarity session

Required safety line:

> Not therapy, diagnosis, or emergency support.

## Screen 2: Onboarding

Collect only:

- country optional
- age band optional
- main reason for visit
- consent checkbox

Show:

> This tool is for reflection and support routing. It is not a crisis service. If you may be in immediate danger, contact emergency services or a crisis helpline now.

## Screen 3: Guided chat

The assistant should collect:

1. what feels off
2. when it started
3. what life areas are affected
4. emotions and body signals
5. what the user has tried
6. current support
7. safety signals

UI elements:

- chat messages
- input box
- visible safety/help button
- progress indicator
- generate clarity map button

## Screen 4: Clarity Map

Sections:

- non-diagnosis notice
- short headline
- main patterns
- focus areas
- next 24 hours
- next 7 days
- suggested support path
- resources
- feedback CTA

## Screen 5: Resources

Show deterministic resources based on:

- country
- risk level
- topic
- support type

## Screen 6: Demo

For judges:

- product journey
- architecture diagram
- safety test matrix
- Codex build log highlights
- live prototype link
